Privacy Policy

Last updated: April 21, 2026

Offline OCR does not collect, transmit, or store any user data. All processing happens entirely within your browser.

Overview

Offline OCR is a Chrome extension that performs Optical Character Recognition (OCR) entirely offline, within your browser. It is designed with privacy as a core principle.

Data Collection

This extension does not collect any data. Specifically:

  • No personal information is collected
  • No images or screenshots are transmitted externally
  • No OCR results or recognized text are sent to any server
  • No usage analytics or telemetry are collected
  • No cookies or tracking mechanisms are used

How It Works

All OCR processing is performed locally using ONNX Runtime Web (WebAssembly). The OCR models (DEIM and PARSeq) are bundled within the extension package and run entirely in your browser. No network requests are made during or after OCR processing.

Permissions

The extension requests the following permissions, all used exclusively for local OCR functionality:

  • scripting — To inject the selection overlay UI into the active tab
  • offscreen — To run the OCR inference engine (ONNX Runtime Web cannot run in a service worker)
  • clipboardWrite — To copy recognized text to your clipboard
  • contextMenus — To add a right-click menu option for launching OCR
  • storage — To save user-configured cleaning rules locally
  • Host permissions — To capture screenshots and inject the selection UI on any page

Data Storage

The extension stores the following data locally on your device only:

  • Cleaning rules — User-configured regex patterns, stored in chrome.storage.sync
  • Model cache — OCR model files cached in IndexedDB for faster loading

This data never leaves your device and can be cleared by uninstalling the extension.

Third-Party Services

This extension does not use any third-party services, APIs, or analytics platforms.

Changes to This Policy

If this privacy policy is updated, the changes will be posted on this page with an updated date.

Contact

If you have questions about this privacy policy, please open an issue on our GitHub repository.

プライバシーポリシー

最終更新日: 2026年4月21日

オフラインOCR は、ユーザーデータの収集・送信・保存を一切行いません。 すべての処理はブラウザ内で完結します。

概要

オフラインOCR は、ブラウザ内で完全にオフラインで動作する光学文字認識（OCR）Chrome拡張機能です。プライバシーを基本設計方針としています。

データ収集について

本拡張機能はデータを一切収集しません。具体的には：

  • 個人情報の収集はありません
  • 画像やスクリーンショットの外部送信はありません
  • OCR結果や認識テキストのサーバー送信はありません
  • 利用統計やテレメトリの収集はありません
  • Cookieやトラッキング機構は使用しません

動作の仕組み

すべてのOCR処理は、ONNX Runtime Web（WebAssembly）を使用してローカルで実行されます。OCRモデル（DEIMおよびPARSeq）は拡張機能パッケージに内蔵されており、ブラウザ内でのみ動作します。OCR処理中および処理後にネットワークリクエストは行われません。

権限について

本拡張機能は以下の権限を要求します。すべてローカルOCR機能のためにのみ使用されます：

  • scripting — アクティブタブに選択オーバーレイUIを注入するため
  • offscreen — OCR推論エンジンを実行するため（ONNX Runtime WebはService Workerでは動作不可）
  • clipboardWrite — 認識テキストをクリップボードにコピーするため
  • contextMenus — 右クリックメニューにOCR起動オプションを追加するため
  • storage — ユーザーが設定したクリーニングルールをローカルに保存するため
  • ホスト権限 — 任意のページでスクリーンショット取得と選択UI注入を行うため

データの保存

本拡張機能は、以下のデータをお使いのデバイスにのみローカル保存します：

  • クリーニングルール — ユーザーが設定した正規表現パターン（chrome.storage.syncに保存）
  • モデルキャッシュ — OCRモデルファイル（IndexedDBにキャッシュ）

これらのデータはデバイスの外に出ることはなく、拡張機能をアンインストールすることで削除できます。

サードパーティサービス

本拡張機能は、サードパーティのサービス、API、分析プラットフォームを一切使用しません。

ポリシーの変更

本プライバシーポリシーが更新された場合、このページに更新日とともに掲載されます。

お問い合わせ
